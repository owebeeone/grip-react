      Binding for number inputs with validation, clamping, and empty value handling.

      This hook provides comprehensive binding for number inputs, including validation, range clamping, and handling of empty string inputs. It's particularly useful for numeric form fields that need validation.

      Parameters

      • grip: Grip<number>

        The Grip to bind to (must be number type)

      • tapGrip: Grip<AtomTapHandle<number>>

        The Grip that provides the AtomTap handle

            parse?: (s: string) => undefined | number;
        }

        Configuration options

        • Optionalclamp?: { max?: number; min?: number }

          Optional min/max values to clamp the input

        • Optionalctx?: GripContext | GripContextLike

          Optional context for data resolution

        • OptionalemptyAs?: number

          Value to use when input is empty (default: undefined)

        • Optionalformat?: (v: undefined | number) => string

          Custom formatting function

        • Optionalparse?: (s: string) => undefined | number

          Custom parsing function

      Returns { onChange: (e: ChangeEvent<HTMLInputElement>) => void; value: string }

      Object with value and onChange props for number input elements

      Example

      const bind = useNumberGrip(USER_AGE, USER_AGE_TAP, {
        emptyAs: 0,
        clamp: { min: 0, max: 120 }
      });

      return <input type="number" {...bind} />;
